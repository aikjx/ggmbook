# 数术工坊业务处理流程规范化方案

## 目标

- 将 `数术工坊` 从“可访问页面”升级为“状态清晰、入口稳定、可持续发布”的专题体系
- 统一卷页、合集页、归档页三类载体的职责，避免用户点击后预期错位
- 建立从内容整理到上线发布的标准流程，保证后续补卷、迁移、归档都能重复执行
- 将“404 修复思路”升级为“入口先稳定、内容再迭代”的长期机制

## 当前业务现状

- 书籍首页：`docs/zh/books/shushu/index.md`
  - 负责数术工坊总入口
  - 目前承载卷册入口、主题介绍、归档入口
- 卷页：`docs/zh/books/shushu/vol1` 到 `vol8`
  - 第 1-3 卷已上线，但正文承载在合集页锚点中
  - 第 4-8 卷为稳定预留页，当前无独立正文
- 合集页：`docs/zh/books/math/articles/161954834/index.md`
  - 当前是数术八卷已上线正文的真实承载体
  - 已具备 `vol1`、`vol2`、`vol3` 锚点
- 归档页：`docs/zh/books/shushu/articles/index.md`
  - 当前为空页
  - 用于承接“数术工坊”专题方向的自动分类外部文章

## 核心问题

- 用户进入首页前，无法区分“已上线正文”“合集跳转”“待上线预留页”
- 卷页、合集页、归档页边界不够显性，容易把三类入口理解成同一种内容
- 当前部分可点击入口只是“技术可访问”，但不一定满足用户“打开即正文”的预期
- 归档页为空时没有被流程化管理，容易让用户误判为坏链或未完成页面
- 后续补卷如果没有统一流程，极易再次出现链接漂移、入口失配和状态不一致

## 业务对象定义

### 1. 书籍首页

- 路径：`docs/zh/books/shushu/index.md`
- 职责：
  - 展示专题定位
  - 展示卷册整体状态
  - 提供快速进入入口
  - 说明当前处理口径
- 禁止事项：
  - 不堆积长正文
  - 不直接承担卷内容的主发布任务

### 2. 卷页

- 路径：`docs/zh/books/shushu/volX/index.md`
- 职责：
  - 每卷必须有稳定入口
  - 展示卷状态、当前载体和下一步动作
  - 若正文尚未独立发布，卷页承担“过渡页”职责
- 强制要求：
  - 每卷都必须存在 `index.md`
  - 即使未上线，也必须有说明文本和返回路径

### 3. 合集页

- 当前路径：`docs/zh/books/math/articles/161954834/index.md`
- 职责：
  - 承接尚未拆卷但已经整理完成的正文
  - 提供卷级锚点跳转
  - 作为阶段性承载体，不替代卷页入口
- 适用场景：
  - 内容已可读，但暂未完成独立卷页拆分

### 4. 归档页

- 路径：`docs/zh/books/shushu/articles/index.md`
- 职责：
  - 承接自动分类进入数术工坊专题的外部文章
  - 作为补充材料，不作为数术八卷主内容入口
- 空状态要求：
  - 必须明确说明“当前无文章”而非简单留空

## 状态模型

卷册状态统一使用以下四档，不允许自定义口径：

1. `已上线`
   - 正文已可读
   - 可以是独立卷页，也可以暂存于合集页
2. `合集承载`
   - 正文已存在，但尚未拆分为独立卷页
   - 首页展示时可并入 `已上线`
3. `待上线`
   - 卷页入口已补齐
   - 仅有稳定落点和说明，不应伪装成已上线正文
4. `归档为空`
   - 仅适用于归档页
   - 表示当前没有命中分类的补充文章

建议展示口径：

- 首页：只向用户显示 `已上线` / `待上线`
- 内部流程：保留 `合集承载` 作为技术状态

## 目录与路由规范

- 首页固定：`docs/zh/books/shushu/index.md`
- 卷页固定：`docs/zh/books/shushu/volX/index.md`
- 归档入口固定：`docs/zh/books/shushu/articles/index.md`
- 内部跳转优先使用相对路径
- 所有对外入口都必须落到真实 `index.md`
- 不允许首页直接指向不存在的文章 ID

## 标准业务流程

### 阶段 1：内容立项

- 明确该内容属于：
  - 数术八卷主线
  - 数术工坊专题归档
  - 其它书籍内容，不应放入 `shushu`
- 若属于数术八卷，先确认对应卷号

### 阶段 2：内容载体判定

- 若正文已完整、但还不适合拆卷：
  - 先挂到合集页
  - 在卷页中配置正文入口和合集入口
- 若正文已完整、且适合独立维护：
  - 直接发布到独立卷页
- 若正文尚未完成：
  - 先创建或维护稳定卷页
  - 标记为 `待上线`

### 阶段 3：入口建设

- 首页新增或更新卷册卡片
- 卷页更新状态、载体和跳转路径
- 如涉及合集正文，必须补锚点
- 如涉及归档文章，更新归档页列表或生成脚本

### 阶段 4：构建前自检

- 自检首页入口是否都能打开
- 自检卷页是否都存在状态说明
- 自检合集锚点是否能正确命中
- 自检归档页空状态或列表状态是否真实

### 阶段 5：构建与验证

执行建议命令：

```bash
pnpm build
```

必要时本地预览：

```bash
pnpm dev --host 127.0.0.1 --port 8081
```

验证项：

- 首页首屏是否能立即看到核心入口
- 卷册卡片状态是否与真实内容一致
- 点击已上线卷册后，是否能抵达正文
- 点击待上线卷册后，是否得到明确说明，而不是空白页
- 归档页为空时，是否显示清晰空状态

### 阶段 6：老师审核

发布前必须经过“乖乖老师”审核，至少检查：

- 是否存在 404
- 公式是否正常渲染
- 文案是否有错别字
- 卷册状态是否真实
- 是否存在“能点开但不是预期内容”的误导入口

### 阶段 7：上线与复盘

- 记录本次变更范围
- 记录新增卷册或状态变化
- 记录遗留问题
- 将经验写回 `project_docs/` 或项目记忆

## 页面职责矩阵

| 页面 | 主职责 | 可承载内容 | 不应承担 |
| --- | --- | --- | --- |
| `shushu/index.md` | 总入口 | 状态、入口、主题摘要 | 长正文主发布 |
| `shushu/volX/index.md` | 单卷稳定入口 | 状态、跳转、独立正文 | 其它卷内容 |
| `math/articles/161954834/` | 阶段性合集承载 | 已上线正文与锚点 | 永久替代卷页入口 |
| `shushu/articles/index.md` | 归档补充 | 自动分类文章 | 数术八卷主内容 |

## 页面展示规范

### 首页

- 必须前置统计信息
- 必须前置卷册入口
- 必须展示每卷状态
- 必须将“合集页”和“归档页”区分开来

### 卷页

- 必须包含：
  - 当前状态
  - 当前载体
  - 正文入口或后续说明
  - 返回首页入口

### 归档页

- 有文章时显示列表
- 无文章时显示空状态说明
- 不允许只有标题没有说明

## 自动化建议

后续建议补以下自动化能力：

- 为 `shushu` 增加专门的状态索引生成脚本，避免手工维护 8 卷状态
- 从卷页正文或元数据自动汇总“已上线 / 待上线”数量
- 为合集页锚点做一致性检测，避免卷页指向失效锚点
- 为归档页增加“空状态 / 数量变化”校验

## 执行清单

每次处理数术工坊内容时，按以下顺序执行：

1. 判定内容归属：卷页、合集还是归档
2. 更新状态：已上线、合集承载、待上线、归档为空
3. 更新入口：首页、卷页、归档页至少同步一处
4. 校验落点：保证所有入口都落到真实页面
5. 本地构建：确认无新增报错
6. 老师审核：检查 404、公式、错别字、状态真实性
7. 发布复盘：记录本轮变更和剩余缺口

## 当前建议落地顺序

1. 先稳定首页与卷页状态展示
2. 再补齐第 4-8 卷的真实正文
3. 然后决定是否把第 1-3 卷从合集页拆为独立卷页
4. 最后再扩充数术工坊专题归档规则
